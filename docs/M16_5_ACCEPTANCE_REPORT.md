**Status:** ✅ Verified

**Changes:**
- `backend/src/domain/discussion.ts` — Added `"stopped"` to `DiscussionStatus`
- `backend/src/routes/discussion.ts` — Added `POST /:id/stop` endpoint (validates discussion exists, is active; sets status to stopped)
- `backend/src/services/DiscussionEngine.ts` — Changed stop check from `=== "finished"` to `!== "active"` (now respects stopped)
- `backend/src/controllers/DiscussionSessionController.ts` — Updated loop check to `status !== "active"`
- `frontend/src/types/discussion.ts` — Added "stopped" to `DiscussionStatus`, updated `toDisplayStatus()`
- `frontend/src/api/discussionApi.ts` — Added `stopDiscussion()`
- `frontend/src/pages/DiscussionRoomPage.tsx` — Added stop button (visible during RUNNING), "stopped" state handling
- `frontend/src/pages/DiscussionRoomPage.module.css` — Added `.stopButton` and `.stoppedBadge` styles

**Discussion states:**
| UI State | Backend Status | Behavior |
|---|---|---|
| `idle` | `active` (no messages) | Start button visible |
| `running` | `active` (has messages) | ON AIR + Stop button |
| `stopped` | `stopped` | "讨论已停止" badge, no messages generated |
| `finished` | `finished` | "讨论已结束" badge |

**Stop mechanism:**
- Frontend sends `POST /stop` → backend sets `status = "stopped"`
- `DiscussionEngine` checks `status !== "active"` before each round → stops
- `DiscussionSessionController` checks `status !== "active"` before each intervention → stops
- After stop: no further messages can be generated

## 6. Known Limitations

### 6.1 Polling Still Temporary
Insights are fetched via the same 2-second polling mechanism as messages. Future: SSE streaming.

### 6.2 Insight Analysis Cost
Each insight poll calls the AI (unless transcript is unchanged). With 2-second polling during a discussion, this generates significant AI calls. **Recommendation:** Add caching in `InsightAnalyzer` (skip analysis if transcript hash unchanged) before production.

### 6.3 Moderator Intervention Context Window
Interventions pass the last 12 messages as context. Very long discussions may need a sliding window strategy with summarization.

### 6.4 No Intervention on First Batch
If `maxRounds` is small (1-2), no interventions fire. This is by design — interventions only trigger when there are enough rounds to batch (BATCH_SIZE=2).

### 6.5 Frontend Insights During Idle
The frontend does not fetch insights while `executionState === "idle"`. Insights only load once the discussion starts. After the discussion ends (stopped/finished), polling stops — the last fetched insights remain visible.
