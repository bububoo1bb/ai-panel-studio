# AI Panel Studio - Acceptance Criteria

## AC-01 首页与讨论列表

- [ ] 首页能够展示已有讨论
- [ ] 用户能够创建新讨论
- [ ] 用户能够进入已有讨论
- [ ] 不同讨论可以独立存在

## AC-02 创建讨论

- [ ] 用户能够输入讨论主题
- [ ] 用户能够指定专家人数
- [ ] 默认专家人数符合需求
- [ ] 提交后进入嘉宾生成流程

## AC-03 嘉宾生成

- [ ] 系统根据主题动态生成 1 名主持人
- [ ] 系统生成指定数量的专家
- [ ] 每位专家具有姓名
- [ ] 每位专家具有职业或 Title
- [ ] 每位专家具有明确立场
- [ ] 每位专家具有专属视觉标识
- [ ] 用户确认阵容后才进入演播厅

## AC-04 圆桌讨论

- [ ] 主持人能够自然开场
- [ ] 主持人能够追问和推进讨论
- [ ] 专家根据角色与上下文发表观点
- [ ] 专家能够补充、质疑或反驳其他观点
- [ ] 发言顺序不是固定机械轮流
- [ ] 单次发言保持简洁
- [ ] 主持人能够自然结束讨论

## AC-05 专家状态

- [ ] 每位专家具有独立状态区域
- [ ] 能够显示待机、准备发言、发言中等状态
- [ ] 能够展示当前关注点或公开思考摘要
- [ ] 不展示模型完整内部 Chain-of-Thought

## AC-06 Transcript

- [ ] 发言实时显示
- [ ] 显示发言人姓名
- [ ] 显示职业或 Title
- [ ] 使用与角色一致的视觉标识
- [ ] 不显示内部调度事件

## AC-07 共识与分歧

- [ ] 讨论过程中能够形成共识信息
- [ ] 讨论过程中能够形成分歧信息
- [ ] 共识与分歧能够随讨论实时更新
- [ ] 不是只在讨论结束后一次生成

## AC-08 讨论总结

- [ ] 讨论结束后生成最终结果
- [ ] 主持人具有自然语言收尾
- [ ] 总结以用户可读方式展示
- [ ] 页面不直接展示内部 JSON

## AC-09 数据与并行讨论

- [ ] 数据使用 SQLite 持久化
- [ ] 页面刷新后核心讨论数据能够恢复
- [ ] 多个讨论之间 Transcript 相互隔离
- [ ] 多个讨论之间状态和事件流相互隔离

## AC-10 实时与安全

- [ ] 使用 SSE 或 WebSocket 实现实时更新
- [ ] 大模型 API Key 仅由后端读取
- [ ] API Key 不出现在浏览器代码或请求响应中

## AC-11 UI

- [ ] 页面主要 UI 使用中文
- [ ] 普通桌面尺寸能够正常使用
- [ ] 主要区域具有合理的独立滚动
- [ ] 不依赖整个页面无限向下滚动完成核心操作
